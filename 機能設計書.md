# Comic Viewer 機能設計書

- 作成日: 2026-07-03
- 対象バージョン: 現行開発版（Windows / Linux 共通ソースベース）
- リポジトリ: https://github.com/makushimasu/comic-viewer

---

## 1. 製品概要

### 1.1 コンセプト
ZIP / RAR アーカイブや画像フォルダを「本棚」形式で管理し、快適に閲覧できる
コミック・画像ビューア。Pico Viewer にインスパイアされた、必要十分な機能を
持つフリーソフトを目指す。

### 1.2 動作環境

| 項目 | Windows | Linux |
|---|---|---|
| OS | Windows 10 / 11 | Ubuntu 22.04 / Mint 21 以降 |
| ランタイム | Python 3.10+（配布版は PyInstaller 単体EXE） | Python 3.10+ |
| GUI | PySide6 (Qt6) | 同左 |
| 画像処理 | Pillow | 同左 |
| RAR 展開 | libarchive または 7-Zip | libarchive または unar / unrar |

### 1.3 モジュール構成

| ファイル | 役割 |
|---|---|
| `main.py` | 本棚ウィンドウ（BookshelfWindow）、ワーカー群、アイテムデリゲート、エントリポイント |
| `viewer.py` | ページビューア（ViewerWindow）、進捗・しおり・回転の永続化 |
| `core.py` | サムネイル生成、thumb_cache 書き込み、プレースホルダー生成 |
| `archive.py` | ZIP / RAR 展開（libarchive ctypes + 外部コマンドフォールバック） |
| `page_cache.py` | 全ページキャッシュ（LRU 削除、上限 MB 設定可） |
| `settings.py` | 設定の読み書き・設定ダイアログ |
| `i18n.py` | 日本語 / 英語の文言リソース |
| `utils.py` | 自然順ソート、ファイル名解析 |
| `wood_bg.py` | 木目テクスチャ生成 |
| `appdir.py` | OS 別データ保存先の解決 |

### 1.4 データ保存場所

| OS | パス |
|---|---|
| Windows | `%LOCALAPPDATA%\comic_viewer\` |
| Linux | `~/comic_viewer/` |

| ファイル / フォルダ | 内容 |
|---|---|
| `library.json` | 登録済みフォルダ / ファイルのパスリスト |
| `progress.json` | ページ進捗・しおり（ページ番号→ラベル辞書）・回転角度 |
| `settings.json` | ユーザー設定 |
| `last_location.json` | 最後に開いたフォルダ / ファイル |
| `thumb_cache/` | 表紙サムネイル（パスの MD5 → `.jpg`、失敗は `.noimg` マーカー） |
| `page_cache/` | 全ページキャッシュ（MD5 ディレクトリ + `meta.json`、mtime 検証付き） |

## 2. 実装済み機能仕様

### 2.1 本棚（BookshelfWindow）

#### 2.1.1 ライブラリ管理
- フォルダ / ファイル（ZIP・RAR・画像）を登録し、`library.json` に永続化する
- 登録アイテムはサムネイル付きグリッド（`WoodListView` = QListView IconMode）で表示する
- フォルダはダブルクリックで中へ移動（階層ブラウズ）。ツールバーの「上へ」「ホーム」で戻る

#### 2.1.2 サムネイル表示
- 表紙サムネイルは `thumb_cache/` に 400×600 で保存し、表示時に設定サイズ（既定 190×270）へ縮小する
- **ウィンドウ表示ロード**: スクロール位置 ±50 件のみ QPixmap を保持し、範囲外は解放する（メモリ上限対策。2000 冊規模でも約 200MB 以内）
- スクロール停止 300ms 後（`scroll_stabilized` シグナル）に表示範囲を再ロードする
- グリッドはビューポート幅に合わせて `gridSize` を動的計算し、右側に無駄な余白を作らない
- サムネイル生成失敗時は「No image」プレースホルダーを表示し、`.noimg` マーカーで再試行を防ぐ

#### 2.1.3 サムネイル生成（ThumbnailWorker）
- QObject + QThread + ThreadPoolExecutor による非同期生成
- 優先順: ①キャッシュ済み（8 並列）→ ② ZIP 未キャッシュ（4 並列）→ ③ RAR 未キャッシュ（2 並列）
- フォルダ移動ごとに `_view_generation` を加算し、古いワーカーの結果を世代番号で破棄する
- ワーカー停止は非ブロッキング（`_soft_stop_thumbnail_worker`）。停止中スレッドは `_zombie_threads` で参照を保持し自然終了を待つ（強制 terminate はクラッシュ要因のため禁止）

#### 2.1.4 検索
- キーワード検索（サブフォルダ再帰、SearchWorker で非同期実行）
- 対象: フォルダ名 / 対応拡張子のファイル名の部分一致

#### 2.1.5 状態復元
- 最後に開いた場所・スクロール位置を記憶し、次回起動時に復元する
- 復元完了後にウィンドウを表示する（ちらつき防止）

#### 2.1.6 しおり・進捗バッジ
- 各アイテムに読書状態を表示: 赤しおり = 読みかけ、青しおり = 読了
- ページキャッシュ済みアイテムにはキャッシュ済みフラグを表示する

### 2.2 ビューア（ViewerWindow）

#### 2.2.1 表示モード
- 1 ページ表示 / 見開き 2 ページ表示
- 読む方向: 右→左（日本式）/ 左→右
- フィットモード: 全体 / 横幅 / 縦幅（`F` キーで切替）
- 全画面表示
- ページ回転: 90° 単位、ページごとに `progress.json` へ永続化
- 埋め込みモード（`viewer_mode: "inline"` — QStackedWidget で本棚内に表示）/
  独立ウィンドウモード（`"window"`）

#### 2.2.2 操作
| 操作 | 動作 |
|---|---|
| `→` / `D` / `Space` | 次のページ |
| `←` / `A` | 前のページ |
| `+` / `-` | ズームイン / アウト |
| マウスホイール | ズーム |
| `F` | フィットモード切替 |
| `Esc` | 本棚へ戻る / 全画面解除 |

#### 2.2.3 ページ読み込み（PageLoadWorker）
- アーカイブ展開は非同期実行。ZIP はストリーミング読みで先頭ページから即表示
- ページキャッシュがあるときは `archive_cached` タイプとして **bytes を RAM に展開せず**
  キャッシュファイルのパス参照で表示する（省メモリ）
- 初回展開時に全ページを `page_cache/` へ JPEG で保存（上限 MB 超過時は atime 順に削除）

#### 2.2.4 しおり・スライドショー
- ページにラベル付きしおりを設定できる（複数可、`progress.json` に保存）
- サムネイルストリップ: 下部にページ一覧を表示、クリックで移動
- スライドショー: 秒数・終了時動作・切替エフェクトを設定可能
- 進捗保存: 最後に読んだページを次回開いたときに復元する

### 2.3 アーカイブ対応（archive.py）

| 形式 | 拡張子 | 展開手段（優先順） |
|---|---|---|
| ZIP | `.zip` `.cbz` | Python 標準 zipfile（外部依存なし） |
| RAR | `.rar` `.cbr` | ① libarchive（RAR4/RAR5 判別）② 7z / unar / unrar コマンド |
| 画像フォルダ | `.jpg` `.jpeg` `.png` `.webp` `.gif` `.bmp` | 直接読み込み |

- サムネイル生成時は先頭 1 枚のみ抽出する（`read_cover`）
- libarchive で読めない RAR はセッション中メモリキャッシュして再試行しない
- ZIP のファイル名は CP932 / UTF-8 を自動判別（文字化け対策）

### 2.4 設定（settings.py）
- サムネイルサイズ、ビューアモード、階層表示、ページキャッシュ上限 MB、言語（日 / 英）ほか
- 設定ダイアログから変更、`settings.json` に即時保存

### 2.5 配布（Windows）
- PyInstaller によるビルド（`build.bat` で一括実行）
  - `comic_viewer.spec` → `dist/ComicViewer/`（本体）
  - `uninstaller.spec` → `dist/uninstall.exe`
  - `installer.spec` → `dist/ComicViewerSetup.exe`（本体 + アンインストーラー同梱、UAC 昇格付き）
- UPX 圧縮は AV 誤検知対策のため**無効**（`upx=False`）
- コード署名: 未対応（signtool 導入済み。証明書取得後に `build.bat` へ署名ステップを追加予定）
- アプリアイコン: `icon.png` / `icon.ico`。タスクバー ID は `ComicViewer.App`
  （SetCurrentProcessExplicitAppUserModelID）

## 3. 拡張機能仕様（未実装・優先度順）

> 2026-07-03 実施の類似ソフト調査（Pico Viewer / NeeView / YACReader /
> CDisplayEx / Honeyview / ZipPla）に基づく。

### 3.1 PDF 対応 【優先度: 最高】

**目的**: 自炊ユーザーの主要フォーマットである PDF を開けるようにする。
Pico Viewer 代替の必須条件。

| 項目 | 内容 |
|---|---|
| ライブラリ | `pypdfium2`（BSD ライセンス、配布に制約なし）を第一候補とする |
| 対応拡張子 | `.pdf` を `SUPPORTED_EXTS` に追加 |
| 変更ファイル | `archive.py`（`read_pdf_all_with_names` / `read_pdf_cover` 追加）、`main.py`（拡張子追加）、`viewer.py`（ページタイプ `"pdf"` 追加）、`requirements.txt` |
| レンダリング | ページ→ラスタライズ（長辺 2000px 目安）→ 既存 page_cache に JPEG 保存。2 回目以降は `archive_cached` 経路をそのまま流用 |
| サムネイル | 1 ページ目をラスタライズして thumb_cache へ |
| 留意点 | pypdfium2 未インストール環境では PDF を一覧に表示しない（graceful degradation） |

### 3.2 横長ページの自動分割 【優先度: 高】

**目的**: 「見開き 2 ページが 1 枚の横長画像」になっている自炊本を、
自動で左右 2 ページに分割して表示する。

| 項目 | 内容 |
|---|---|
| 判定 | `幅 > 高さ × 1.15` を見開き画像とみなす（閾値は定数化） |
| 分割順 | 読む方向が右→左なら右半分→左半分、左→右なら逆 |
| 変更ファイル | `viewer.py`（ページリスト構築時に論理ページへ展開。`_get_pixmap` で半分を crop）、`settings.py`（ON/OFF 設定、既定 ON） |
| 影響範囲 | ページ番号・進捗・しおりは論理ページ基準になるため、保存時は「物理ページ + 左右」で記録する |

### 3.3 ドラッグ＆ドロップ登録 【優先度: 高（最小工数）】

**目的**: エクスプローラー / ファイラーからフォルダ・アーカイブをドロップ
するだけで本棚に登録できるようにする。

| 項目 | 内容 |
|---|---|
| 変更ファイル | `main.py` のみ |
| 実装 | `WoodListView.setAcceptDrops(True)`、`dragEnterEvent`（URL かつ対応拡張子 / フォルダなら accept）、`dropEvent`（既存の登録処理 `registered_items` 追加 + `save_library()` + `refresh_view()` を呼ぶ） |
| 挙動 | ルート本棚表示中のみ登録扱い。複数ドロップ対応。重複は無視して件数を通知 |

### 3.4 7z / cb7 対応 【優先度: 中】

| 項目 | 内容 |
|---|---|
| 展開手段 | libarchive は 7z 対応済み → 既存コードの形式フィルタ追加が主作業。フォールバックは既存の 7-Zip / unar 経路を流用 |
| 変更ファイル | `archive.py`（`.7z` `.cb7` 分岐追加）、`main.py`（`SUPPORTED_EXTS` 追加） |

### 3.5 閲覧履歴 【優先度: 中】

| 項目 | 内容 |
|---|---|
| データ | `history.json`（新規）: `[{path, opened_at}]` 最大 30 件、開くたびに先頭へ |
| UI | ツールバーに「履歴」メニュー（QMenu）。クリックで該当ファイルをビューアで開く。存在しないパスはグレー表示 |
| 変更ファイル | `main.py`（メニュー + 保存処理。`_open_viewer` にフック） |

### 3.6 画像フィルタ（明るさ・グレースケール・背景色） 【優先度: 中】

**目的**: 黄ばんだ自炊本の白色補正、夜間閲覧の減光。Pico Viewer の主要機能。

| 項目 | 内容 |
|---|---|
| フィルタ | 明るさ（0.5〜1.5）/ コントラスト / グレースケール ON/OFF |
| 実装 | `viewer.py` の QImage 変換直前に PIL `ImageEnhance` を適用。フィルタ変更時は表示中ページのみ再処理（キャッシュには適用しない） |
| 背景色 | ビューア背景色を設定ダイアログから変更可能にする（既定: 現行の木目 / 黒） |
| 変更ファイル | `viewer.py`、`settings.py`、`i18n.py` |

### 3.7 余白自動カット 【優先度: 中】

| 項目 | 内容 |
|---|---|
| 実装 | PIL `ImageOps.invert` + `getbbox()` で非白領域を検出し crop。閾値（ほぼ白 = 245 以上等）は定数化 |
| ばらつき対策 | 開いている本の先頭 10 ページから共通カット幅を推定し、全ページに同一 crop を適用する（ページごとにガタつかせない） |
| 変更ファイル | `viewer.py`（表示パイプラインに挿入）、`settings.py`（ON/OFF、既定 OFF） |

### 3.8 縦スクロールモード（ウェブトゥーン） 【優先度: 低】

| 項目 | 内容 |
|---|---|
| 実装 | 現行のページ単位表示とは別系統の表示モード。QScrollArea + 遅延ロード（可視範囲 ± 2 ページのみ QPixmap 保持 — 本棚のウィンドウ表示ロードと同じ考え方） |
| 変更ファイル | `viewer.py`（大規模改修）、`settings.py` |
| 備考 | 実装コストが高いため、上位機能が揃ってから着手する |

### 3.9 キーカスタマイズ 【優先度: 低】

| 項目 | 内容 |
|---|---|
| 実装 | `settings.json` に `keymap` 辞書（アクション名 → キーシーケンス）。`viewer.py` の keyPressEvent をアクションテーブル参照に書き換える |
| UI | 設定ダイアログに割り当てタブを追加 |

### 3.10 ルーペ・マウスジェスチャー 【優先度: 低】

- ルーペ: マウス位置周辺を円形 / 矩形で拡大表示（QPainter オーバーレイ）
- ジェスチャー: 右ドラッグの軌跡でページ送り等
- いずれも NeeView 系の高機能路線。ニーズを見て判断する

## 4. 非機能要件（現行実装で担保済み）

| 項目 | 内容 |
|---|---|
| メモリ | 大規模本棚（2000 冊）でも常駐約 200MB 以内（ウィンドウ表示ロード + 表示サイズ縮小 + パス参照キャッシュ） |
| 応答性 | 重い処理はすべてワーカースレッド。UI スレッドをブロックしない |
| スレッド安全性 | ワーカー停止は非ブロッキング。`thread.terminate()` は使用禁止 |
| ネットワークドライブ | 未接続時の Windows エラーダイアログを抑制（SetErrorMode）。切断時もクラッシュしない |
| 文字コード | ZIP 内ファイル名の CP932 / UTF-8 自動判別 |
| AV 誤検知対策 | UPX 無効。コード署名は証明書取得後に対応 |
| 多言語 | i18n.py による日 / 英切替 |
